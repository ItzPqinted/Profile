How is this website created? Who hosts it? What programming languages are used?

You will get to know all about it below!

First things first, let's talk about the programming languages used here.

For this website, the main language used here is HTML 5 obviously. It's styled by CSS 3. JavaScript is used for scroll-to-top function. Games here are created by JavaScript with P5.JS Library along with Matter.JS Library.
In Online games, Google's Firebase Console is used for realtime synchronisation.

How is this website hosted?

This website is... I won't call it 'hosted' but, let me simplify this a bit.
As you can see, in the link address there's a domain GitHub. Check it out Github Community.
The files of this website are commited and uploaded to GitHub and GitHub Pages are used to share this output as a share-able link. As simple as that, only those who have this link can view this website. You won't be able to search this website on google, there's no such possible way.
Obviously, anyone can do this.

Can you tell me more about this website?

This website is quite Static.
There is no such back-end thing here. So, your privacy is protected. No one (even me) know(s) who checked this website out. But, here in Online Games the information you are supposed to specify can be read by me obviously, as I have to reset infromation after playing and I have to monitor my database once a week. Rest of the things, I have no idea who's viewing this website. Hence, loading is much more faster (if you've noticed).
This Page or the Index is the homepage, while this is a webpage.



Some Statistics about this website
This website took about 7 days (average 5-6 hours per day) for this over-all output. Structuring took about 3 days, while styling and additional adjustments and accessories took about other 3-4 days.
I started developing this on 30th April, 2021 and commited it to Github at 5th April, 2021, well, I didn't release the Mobile Browser Support during that time.
7th April was the date when Mobile Browser Support was released. Why so much time?
Mobile and Desktop screens are different in dimensions obviously, so some things are misplaced when viewed on mobile. So, I had to implement RWD (Responsive Web Design) to make it mobile-compatible.
